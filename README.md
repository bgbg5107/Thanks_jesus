# 감사 기록 앱 🌿

기획서(감사기록앱_기획서.docx) 기반으로 개발된 묵상 중심 감사 기록 PWA입니다.
Phase 1~3 전체 기능이 포함되어 있습니다.

## 기술 구성

- 프론트엔드: React + Vite, PWA(홈 화면 설치·오프라인 지원)
- 백엔드: Supabase (인증 · PostgreSQL · 스토리지)
- 디자인: 딥 그린/세이지/웜 그레이/베이지, 명조체(Noto Serif KR), 300~500ms 느린 트랜지션, 다크모드

## 처음 실행하기 (약 10분)

### 1. Supabase 프로젝트 만들기

1. https://supabase.com 에서 가입 후 **New Project** 생성 (무료 플랜 가능)
2. 대시보드 → **SQL Editor** → `supabase/schema.sql` 파일 내용 전체를 붙여넣고 **Run**
3. 대시보드 → **Settings → API** 에서 `Project URL`과 `anon public key` 복사

### 2. 환경 변수 설정

`.env.example`을 복사해 `.env` 파일을 만들고 값을 채웁니다.

```
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJ...
```

### 3. 실행

```bash
npm install
npm run dev        # 개발 서버 (http://localhost:5173)
npm run build      # 배포용 빌드 (dist/)
```

배포는 Vercel/Netlify에 dist를 올리거나 저장소를 연결하면 됩니다.
(환경 변수 2개를 배포 설정에도 등록하세요)

### 4. 관리자 계정 지정

1. 앱에서 관리자용 계정으로 회원가입
2. Supabase SQL Editor에서 이메일만 바꿔 실행:

```sql
update public.profiles set is_admin = true
where id = (select id from auth.users where email = '관리자이메일@example.com');
```

3. 설정 탭에 '관리자 페이지' 링크가 나타납니다.

> 기획서의 고정 ID/PW(intouch/8291) 방식 대신, 기획서 보안 권고에 따라
> 계정 플래그 방식을 사용했습니다. 비밀번호는 Supabase가 bcrypt로 해시 저장합니다.

### 5. (선택) 이메일 인증 끄기

테스트를 빠르게 하려면 Supabase 대시보드 → Authentication → Providers → Email
에서 **Confirm email**을 꺼두면 가입 즉시 로그인됩니다.

## 기능 지도

| 탭 | 기능 |
|---|---|
| 오늘 | 오늘의 말씀(자정 기준 자동 갱신) · 감사 항목 여러 개 이어쓰기 · 사진 여러 장 · 감정 11종 · 공유 범위(나만/셀/개인) · 저장 시 말씀 팝업 · 오프라인 작성 후 자동 동기화 |
| 달력 | 날짜별 감정 이모지 · 월 통계 · 감정 분포 · 연속 기록일(스트릭) · 기간 선택 PDF(인쇄)/텍스트 내보내기 |
| 나눔 | 공유된 기록 피드 · 좋아요 · 신고 · 차단 |
| 셀 | 셀 생성(생성자=팀장) · 이름 검색 초대(동명이인 A/B 자동 구분) · 수락/거절 · 팀장 방출 |
| 알림 | 셀 초대 수락/거절 · 좋아요 · 관리자 경고 |
| 설정 | 라이트/다크/시스템 연동 · 매일 리마인드 알림 · 차단 관리 · 로그아웃 · 계정 탈퇴(전체 데이터 영구 삭제) |
| 관리자 | 신고 처리(경고/게시물 삭제/기각) · 유저 정지/해제 · 비밀번호 재설정 링크 발송 |

## 구현 참고 사항

- **동명이인 처리**: 첫 가입자는 이름 그대로, 같은 이름이 또 가입하면 기존 사용자는 자동으로 `이름A`, 새 사용자는 `이름B`가 됩니다 (DB 트리거).
- **성경 말씀**: 기획서 권고대로 외부 API 없이 개역개정 감사 구절 50개를 자체 DB에 시드했습니다. 구절은 로컬에도 캐시되어 오프라인에서도 표시됩니다. (개역개정 저작권은 대한성서공회에 있으며, 상업 배포 시 이용 허락 확인이 필요합니다)
- **리마인드 알림**: 웹 특성상 앱(탭)이 열려 있을 때 브라우저 알림으로 동작합니다. 백그라운드 푸시(FCM)는 서버 구성이 필요해 추후 확장 항목입니다.
- **오프라인**: 네트워크가 없으면 기록이 기기(IndexedDB)에 보관되고, 연결이 복구되면 사진 포함 자동 업로드됩니다.
- **하루 1건 규칙**: `entries`에 `unique(user_id, date)` 제약으로 보장되며, 같은 날 저장은 덮어쓰기(upsert)됩니다.
