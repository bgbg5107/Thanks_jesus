# Memory.md — 지영님의 개발 방향·스타일 메모

이 파일은 개발 과정에서 반복된 피드백과 선호를 기록한 것이다.
새 기능·수정 작업 시 **요청에 없더라도 아래 원칙을 기본으로 반영**할 것.

## 1. UX 철학: "버튼보다 자동으로, 조용하게"

- **자동 저장이 기본.** 명시적 저장 버튼은 최소화한다.
  - 패턴: 첫 저장만 버튼(의식적 행위, 말씀 팝업 동반) → 이후 변경은 3초 디바운스 자동 저장.
  - 자동 저장 시 `.notice` 스타일(세이지 배경·가운데 정렬)로 "자동으로 저장되었습니다." 표시.
  - 팝업을 닫을 때 미저장 변경분은 조용히 저장(flush)하고 닫는다.
  - **편집 팝업이 열려 있는 동안은 자동 저장 보류** — 저장이 빈 항목을 걷어내며
    편집 중인 카드가 사라지는 사고 방지 (홈 editingIdx 가드).
  - 내용을 모두 지우면 별도 버튼·확인 없이 기록 자체가 조용히 비워진다 (같은 3초 호흡).
- **알림창·알림 문구는 문장 단위로 줄바꿈.** 두 문장 이상이면 `\n`으로 나눈다 —
  물음과 부연(경고·안내)을 한 줄에 붙이지 않는다. Dialog·`.notice`·`.error`·알림 목록은
  모두 `white-space: pre-line`이라 `\n`이 그대로 줄바꿈된다.
  예: "…을(를) 없앨까요?\n되돌릴 수 없습니다.", "이름1, 이름2, 이름3\nN명을 셀리더 해제할까요?"
- **보기/수정 모드를 나누지 않는다.** 열면 바로 고칠 수 있어야 한다 ('수정하기' 버튼 단계 제거됨).
- **중복 UI를 싫어한다.** 같은 정보(하트 2개, 캡슐+필터탭 등)를 두 번 보여주지 않는다.
- **보편적 패턴을 따른다.** 망설여지면 인스타그램/카카오톡의 익숙한 배치를 참고
  (예: 날짜는 작성자 이름 아래 작은 글씨, 좋아요 명단은 "N명이 마음을 전했어요" 문구 클릭).

## 2. 모바일 우선 — 레이아웃 깨짐에 민감

- 주 사용 환경은 **모바일**. 모든 UI는 좁은 화면에서 먼저 확인한다.
- 버튼 글자는 절대 줄바꿈되지 않게 (`white-space: nowrap`), 넘치면 크기 축소나 2×2 배치로.
- 요소가 화면/카드 밖으로 넘치는 것(`min-width` 문제 등)을 반복 지적받았음 — 수정 후 좁은 폭 기준으로 점검.
- 현재 상태가 눈에 보여야 함: 활성 탭은 채움 배경, 미확인 알림·새 나눔은 빨간 숫자 배지(탭 열면 자동 해제).

## 3. 입력 UX 디테일

- 감사 항목은 메모장식 불릿 편집(`ItemsEditor` 공용 컴포넌트 사용):
  Enter=새 불릿, 빈 항목 Backspace=불릿 삭제, **한글 IME 조합 중 키 이벤트는 무시**(글자 밀림 버그 방지).
- 항목 내 줄바꿈은 표시 시 그대로 유지(`white-space: pre-line`) + 불릿 기준 들여쓰기 정렬 + 항목 사이 점선 구분.
- 감정 선택은 접기/펼치기 토글(선택 시 자동 접힘), 12종 4열 유지.
- 이름 검색은 **쉼표/공백으로 여러 명 동시 검색** 지원이 기본 (관리자·초대·공유 모두).
- 검색 후 연속 작업 시 검색어를 지우지 않는다 (여러 명 연속 초대 등).
- 사진: 고르면 즉시 썸네일 표시, JPEG 변환+1600px 축소(HEIC 대비), 클릭 시 확대(Lightbox 공용 컴포넌트).
- 브라우저 기본 검증 말풍선 금지 — 앱 스타일 안내 문구로 (라벨에 조건 명시, 예: "비밀번호 (6자리 이상)").

## 4. 도메인 규칙 (교회 공동체)

- **셀**(kind=cell): 교회 공식 모임. 셀리더 권한자(관리자가 지정)만 생성. 이름은 '~셀' 접미사 자동 통일.
- **나눔 공동체**(kind=group): 누구나 생성하는 자유 모임. 이름 규칙 없음.
- 역할 호칭: 셀은 셀리더/셀원, 나눔 공동체는 모임지기/멤버.
- 공유 카테고리는 4개: 나만 보기 / 셀원들과 공유 / 나눔 공동체와 공유 / 개인 공유 (DB는 'team' 하나로 통합).
- **프라이버시 중시**: 관리자도 남의 기록을 볼 수 없다(신고된 게시물만 예외). 나눔 대상 표시도 노출 최소화.
- **상대방 배려**: 초대 1분 내 취소 시 알림까지 조용히 회수. 자기 글 좋아요는 자신에게 알림 안 감.
- 좋아요 수락 등 상호작용은 상대에게 알림으로 이어져야 함 (초대 수락 → 리더에게 알림).
- 나눔 피드는 **당일 우선**: 오늘 나눔만 바로 보이고, 지난 나눔은 최근 7일까지만 토글로 접어서 제공.

## 5. 색·디자인 방향

- **2026-07 UI 리뉴얼**: 레퍼런스 기반 "쨍한 색 + 트렌디·젊은 느낌"으로 전환.
  크림 배경 + 라임(말씀)/라벤더(감사)/오렌지(오늘) 포인트, 본문은 에스코어드림(SCDream)
  자체 호스팅(`public/fonts/` woff2 서브셋, 원본 otf는 `font/`), 말씀만 명조 유지.
- **주 버튼·선택 상태는 블랙(`--ink`)** — 담기/기록하기 버튼, 선택된 칩·감정.
  네이비(`--brand`)는 보조 포인트(기록한 날 링, 링크성 텍스트 등)로만. 메인 컬러는 여전히 딥 네이비(공동체 상징색) — 주 버튼·선택 상태.
- 홈은 "안녕하세요, 이름" 인사 + 알림 벨(미확인 배지) + 프로필 사진 + 주간 스트립 +
  섹션 제목(카드 밖) 구조. 알림은 하단 탭에서 빠지고 홈 헤더 벨로만 진입 (하단 탭 5개).
- 홈 맨 아래 '오늘의 메모'(`components/MemoCard.jsx`, 홈 목록은 스카이 `--sky` 카드,
  `memos` 테이블): **하루 여러 장**(말씀 메모·할 일 메모 등, id uuid PK — migration-13).
  홈에는 제목 한 줄 목록(아이폰 메모식) + '+ 새 메모'. 탭하면 **전체 화면 종이 편집**
  (`.memo-sheet` — 흰 배경·상단 뒤로가기·하단 툴바). **제목/부제목 입력은 없음** —
  첫 줄이 제목 역할(`::first-line` 강조, 목록 제목은 본문 첫 24자). DB의
  title/subtitle 컬럼은 옛 메모 표시용으로만 유지. 본문 contentEditable(굵게·형광펜·
  인용·말씀, HTML 저장·sanitize) + 사진 + 지난 메모 목록/보기 모달.
  3초 자동 저장·비우면 조용히 삭제, 오프라인 초안은 localStorage.
- **말씀 담기(개역한글 전권)**: 메모 본문에 툴바 📖(말씀 찾기 팝업, `VersePicker.jsx`)
  또는 "요 3:16" 타이핑 인식 칩으로 구절 삽입(라임 인용 블록 `.vq`, 명조).
  말씀 블록에는 ❝ 인용부호(`.vq::before`) — 말씀임이 한눈에 보이게 (2026-07 요청).
  말씀 찾기는 타이핑 외에 **책→장→절 짚어 고르기**(구약/신약 그리드, 절은 시작→끝
  두 번 탭으로 범위)와 **최근 담은 말씀 8개**(localStorage `verse-recent`,
  `bible.js`의 recentVerses/rememberVerse) 재담기를 지원.
- **메모 본문 DOM 조작은 반드시 execCommand(insertHTML/delete)를 거칠 것** —
  직접 insertNode/remove 하면 브라우저 undo(Cmd+Z) 스택이 끊긴다 (2026-07 지적).
  말씀 블록이 맨 위면 앞에 빈 줄을 자동 확보해 위로 돌아가 적을 수 있게 한다.
- **메모 편집 키보드 대응(iOS)**: 키보드가 떠도 서식 툴바가 보여야 함 —
  `visualViewport`에 맞춰 `.memo-sheet` 높이를 줄이는 방식(MemoCard의 fit 이펙트,
  `kb-open` 클래스). 100dvh만 믿지 말 것.
  본문 데이터는 `public/bible/krv.json`(저작권 만료된 개역한글판, 1189장 31,102절 검증됨)
  — 첫 사용 시 내려받아 IndexedDB 보관(`lib/bible.js`), 합쳐진 절(롬 9:1-2 등)·(없음) 절 처리 포함.
  개역개정은 대한성서공회 저작권 때문에 넣지 않기로 결정(2026-07).
- 여러 버튼 중 하나를 고르는 선택 UI는 **선택지마다 다른 색을 써도 좋다**
  (예: 함께 나누기 4종 — 네이비/그린/바이올렛/오렌지, `--vis-*` 변수).
- 다른 페이지(캘린더·나눔·공동체 등)도 이 톤으로 순차 정리 중.

## 5.5 UI/UX 원칙 (2026-07 전면 정비)

- 상세 규칙은 **CLAUDE.md의 "UI/UX 필수 원칙" 섹션**이 단일 기준 (여기 중복 기재 안 함).
  요지: 타이포 `--fs-*` 토큰만·12px 미만 금지, AA 대비 유지, 터치 44px,
  기능 아이콘은 Icon.jsx(이모지 금지, 장식 이모지는 유지), 팝업은 Overlay.jsx.

## 6. 작업 수칙 (매번 지킬 것)

- 수정 후 반드시 `npm run build` 통과 확인.
- DB를 건드리면 `supabase/migration-N-*.sql` 증분 파일 + `schema.sql` 동시 갱신, 실행 안내까지.
- 모든 문구는 앱 톤(정중하고 사색적, CLAUDE.md '앱 정체성') 유지. 기술적/명령형 문구 금지.
- 색은 CSS 변수만, 트랜지션은 `var(--t)`(경쾌하되 부드럽게) 유지.
